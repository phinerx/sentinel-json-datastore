// Load existing data or initialize new\n\n    // Add a new user\n    const newUser = {\n        id: 'user123',\n        username: 'alice',\n        email: 'alice@example.com',\n        passwordHash: 'hashedpassword123',\n        role: 'user'\n    };\n    await db.insert('users', newUser);\n    console.log('User added securely!');\n\n    // Retrieve data\n    const allUsers = await db.find('users', {});\n    console.log('All users:', allUsers);\n\n    // Update data\n    await db.update('users', { id: 'user123' }, { $set: { email: 'alice.updated@example.com' } });\n    console.log('User email updated!');\n\n    // Delete data\n    await db.remove('users', { id: 'user123' });\n    console.log('User removed!');\n\n    await db.save(); // Persist changes\n}\n\ninitializeDatastore().catch(console.error);\n```\n\n## 📐 Architecture\n\nSentinel JSON Datastore employs a layered architectural approach to ensure both performance and security. Data is encrypted at the application layer before being written to disk, and integrity checks are performed on every read.\n\n![System Architecture](https://placehold.co/800x400/1e293b/38bdf8?text=Sentinel+JSON+Datastore+Architecture)\n\n### Key Components\n\n*   **Core Datastore Engine:** Manages JSON data structures, indexing, and querying.\n*   **Encryption Module:** Handles all cryptographic operations (AES-256 GCM recommended).\n*   **Integrity Verification:** Uses HMACs and checksums to detect tampering.\n*   **Access Control Layer:** Enforces RBAC policies before data operations.\n*   **Transaction Log:** Appends-only log for auditing and recovery.\n\n## 🔒 Security Deep Dive\n\nSecurity is the cornerstone of Sentinel. We implement industry best practices to safeguard your data:\n\n*   **Data at Rest Encryption:** All data files are encrypted using robust algorithms like AES-256 GCM, ensuring that even if the underlying storage is compromised, your data remains unreadable.\n*   **Data in Transit Protection:** While Sentinel primarily focuses on data at rest, it provides hooks and recommendations for secure communication channels (e.g., HTTPS/TLS) when integrated into web applications.\n*   **Key Management:** Supports integration with external Key Management Systems (KMS) for secure key storage and rotation.\n*   **Input Validation:** Comprehensive validation of all incoming data against defined schemas to prevent injection attacks and data corruption.\n*   **Auditing and Logging:** Detailed, immutable audit trails of all data access and modification attempts, crucial for compliance and forensic analysis.\n*   **Threat Modeling:** Continuous threat modeling and security reviews to identify and mitigate potential vulnerabilities.\n\n![Security Model](https://placehold.co/800x400/1e293b/38bdf8?text=Sentinel+Security+Model)\n\n## 📖 API Reference\n\nThe Sentinel Datastore API is designed to be intuitive and powerful.\n\n### `new SentinelDatastore(options)`\n\nInitializes a new datastore instance.\n\n*   `options.filePath` (string, required): Path to the JSON file where data will be stored.\n*   `options.encryptionKey` (string, required): A 32-byte (256-bit) secret key for AES-256 encryption.\n*   `options.schema` (object, optional): JSON schema for data validation.\n\n### `db.load()`\n\nLoads data from the `filePath`. If the file doesn't exist, it initializes an empty datastore.\n\n### `db.save()`\n\nPersists the current datastore state to the `filePath`, encrypting it.\n\n### `db.insert(collectionName, data)`\n\nInserts a new document into the specified collection. `data` must conform to the schema (if provided).\n\n### `db.find(collectionName, query)`\n\nFinds documents in the specified collection matching the `query`. Supports basic query operators.\n\n### `db.findOne(collectionName, query)`\n\nFinds a single document matching the `query`.\n\n### `db.update(collectionName, query, update)`\n\nUpdates documents matching the `query` with the `update` object. Supports `$set`, `$unset`, etc.\n\n### `db.remove(collectionName, query)`\n\nRemoves documents matching the `query`.\n\n## 🤝 Contributing\n\nWe welcome contributions to the Sentinel JSON Datastore!
